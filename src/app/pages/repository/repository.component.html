3. Repository

This means that the preparations have been made to create a repository that provides the basic database functions (including the CRUD methods) for our two classes, Zoo and Animal.

What kind of information do we want to get from a repository besides the basic queries?

For a zoo we will surely want to get a list of all the animals in that zoo and of course know how many animals there are in the zoo.

In addition, we would like to receive a list of animals of a specific species for the selected zoo.

For an animal, we want to know which zoo it belongs to.

In the given example, the repositories are not customized. If this were desired, an interface AnimalRepositoryCustom and/or ZooRepositoryCustom with the associated implementation AnimalRepositoryCustomImpl and/or ZooRepositoryCustomImpl would be created and used accordingly.
