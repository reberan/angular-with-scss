7. Configuration

So that the Spring framework can run and wire everything as desired, the annotations are attached directly to the code and xml configuration files are created.

Spring Config Files
In this example we assume a "Core Service". With a service in the customer project, the beans come in the masterconsole.xml and jobcommander.xml.

The following jump configurations are to be made on the WMS

page.

The local services are defined as beans in the config-wms-ord-zoo-service.xml file. 

 config-wms-ord-zoo-service.xml

In order for the file to be loaded, a corresponding reference must be entered in the config-wms-ord-generalservice.xml file. 

config-wms-ord-generalservice.xml

In the config-wms-ord-remoteservices.xml file, the RMI services are registered as beans. 

 config-wms-ord-remoteservices.xml

And finally, on the client side, the following configurations are made in the config-client-remoteservices-wms.xml file. 

 config-client-remoteservices-wms.xml

All the necessary configurations have now been created and the services can be tested. To use the Remote Services in a distributed environment and not on a PC as in this example, the appropriate entries must be made in the site.properties file.

7.1 Automatic adjustments to the schema

The database schema update is controlled via the bean jpaProperties with the property hibernate.hbm2ddl.auto.

For the CORE, the configuration is in the config-base-database.xml file and validate is the default. To create the scheme, this value must be set to at least update.

7.2 Customizations

A new config-wms-hibernate-PRJ.xml must be created in the project so that the customer-specific repositories can also be found by the JPA.
